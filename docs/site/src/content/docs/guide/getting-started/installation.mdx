Astro Snapshot is available on both [JSR](https://jsr.io/@twocaretcat/astro-snapshot) and [npm](https://www.npmjs.com/package/@twocaretcat/astro-snapshot), so you can choose where you want to install it from. Either way, you get full type support.

:::tip[JSR or npm?]
**JSR** is a good choice if you're using TypeScript or Deno. It ships typed, modern ESM code by default with no need for separate type declarations.

**npm** may be preferable if you're using a Node-based project, as `astro` is resolved as a peer dependency which avoids potential version mismatch issues. It also works seamlessly with `astro add`.
:::

:::note[Some package managers may block Puppeteer's install scripts]
If Puppeteer fails to download a browser during installation, refer to the [Troubleshooting](/guide/troubleshooting) page.
:::

## Automatic (with `astro add`)

You can use the Astro CLI to install and set up the integration automatically using your preferred package manager.

:::note
`astro add` installs from npm. To use the JSR version, follow the manual installation steps below.
:::

<PackageManagerTabs

## Manual

If you run into any issues, you can install the package manually instead.

### 1. Install

First, install the package using your preferred package manager.

	<Fragment slot="npm">
		```bash title="npm"

		```
		```bash title="JSR"

		```

	<Fragment slot="pnpm">

	</Fragment>
	<Fragment slot="bun">

### 2. Add to Config

Then add it to your `astro.config.*`.

```ts title="astro.config.ts"
import { defineConfig } from 'astro/config';
import snapshot from '@twocaretcat/astro-snapshot';// [!code ++]

export default defineConfig({
  integrations: [],// [!code --]
  integrations: [snapshot({ pages: {} })],// [!code ++]
});
```

Next, you'll need to [configure](/guide/getting-started/configuration) some pages to screenshot.
